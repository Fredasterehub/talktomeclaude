What it does

Speak. It works. It talks back.

One local command. The listening runs on your machine, the speaking runs on your machine — Claude's own brain is the only thing left in the cloud.

Hears you

Local speech-to-text, Whisper-class. Your voice never leaves the box. GPU tier if you've got CUDA, CPU tier if you don't.

Answers back

Speaks Claude's actual dialogue in a real voice — and only the dialogue. Tool calls, fenced code, and thinking are stripped before a word is said.

Rides Claude Code

A plugin Stop hook speaks every reply the moment Claude finishes. Async, non-blocking, fails silent — it never gets in Claude's way.

Three ways to talk

Always-on and pause-gated, push-to-talk, or push-toggle. Set your default once and forget it.

A real mute

assist off and the whole thing goes quiet — hook included. A mute switch that actually mutes.

Never a robot on day one

Three public-domain voices, fetched automatically on first use. No copyrighted voices, no celebrity clones — bring your own on top.

The one thing that matters

It speaks the words. Never the code.

The whole trick is the filter. It reads the transcript and speaks Claude's prose — striking out the tool calls, the logs, and every fenced code block before your headphones ever hear them.

Get it running

Three words: get it, build it, install it.

Every platform spells the same three steps differently. Pick yours. The clone is tiny — the voices download once on your first speak (~250 MB, cached), or run voices --download to pre-fetch.

# Windows Terminal · PowerShell # 1 · install uv (it fetches Python for you) PS> powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex" # 2 · grab the code (Code ▸ Download ZIP, or git clone), then step in PS> cd talktomeclaude # 3 · build the env & install PS> uv venv --python 3.12; uv pip install -e . # hear it PS> .\.venv\Scripts\talktomeclaude speak "Hello from Claude." ▸ playing… (voice: en_US-ljspeech-high)
# macOS · Terminal.app # 1 · install uv (fetches Python too) $ curl -LsSf https://astral.sh/uv/install.sh | sh # 2 · clone (tiny — voices download on first speak) — first git run prompts for Command Line Tools $ git clone https://github.com/Fredasterehub/talktomeclaude.git && cd talktomeclaude # 3 · build the env & install $ uv venv --python 3.12 && uv pip install -e . # hear it $ ./.venv/bin/talktomeclaude speak "Hello from Claude." ▸ playing… (no CUDA on a Mac → speech-to-text runs the CPU tier)
# Linux / Unix # 1 · system bits: PortAudio + git $ sudo apt install -y libportaudio2 git # dnf: portaudio · pacman: portaudio # 2 · install uv, then clone (tiny — voices fetch on first use) $ curl -LsSf https://astral.sh/uv/install.sh | sh $ git clone https://github.com/Fredasterehub/talktomeclaude.git && cd talktomeclaude # 3 · build & install (NVIDIA? add: uv pip install -e ".[cuda]") $ uv venv --python 3.12 && uv pip install -e . $ ./.venv/bin/talktomeclaude speak "Hello from Claude." ▸ playing… (voice: en_US-ljspeech-high)
Plug into Claude Code  ·  claude --plugin-dir . to test, or /plugin install . inside Claude Code to keep it. On native Windows the spoken-reply Stop hook is a bash script — install Git for Windows (winget install --id Git.Git -e) so Claude Code routes it through Git Bash. Everything else works either way. First transcription pulls a Whisper model once (~486 MB CPU · ~3 GB GPU), then caches forever.
On a server over SSH? (Proxmox · homelab · VPS)  ·  Install here on the laptop you sit at, install Claude Code on the server — the voice stays local, Claude runs remote. Full setup just below ↓

Remote / SSH

Voice here. Claude there.

Sitting at a laptop but coding on a server — a Proxmox box, a homelab machine, a VPS you SSH into? That server has no microphone and no speakers, so the voice runs on your laptop, Claude runs on the server, and only text crosses the wire. Set it up once and just talk.

# ON THE SERVER — where Claude Code lives (your SSH session) $ claude --version # confirm Claude Code is installed $ claude # run once to log in, then exit — that's all the server needs
# ON YOUR COMPUTER — where the mic + speakers are (after you install talktomeclaude) # 1 · passwordless SSH, so it never stops to ask for a password mid-sentence $ ssh-keygen -t ed25519 # press Enter through the prompts $ ssh-copy-id you@192.168.2.122 # macOS/Linux (Windows one-liner: see README) $ ssh you@192.168.2.122 echo ok # must print ok, with NO password # 2 · point talktomeclaude at the server — just once, it's remembered $ talktomeclaude config set remote you@192.168.2.122 # 3 · talk. mic + reply stay local; Claude runs on the server $ talktomeclaude listen
Back to all-local  ·  talktomeclaude config set remote local. One tool, any setup — everything on a Mac laptop, or laptop→Proxmox. The full walkthrough (including the Windows key-copy one-liner and troubleshooting) is in the README.

The name

Where the name comes from.

"Covey leader to Raven…
talk to me, Johnnie."

The name descends straight from Talk To Me Johnnie, John Welbourn's old strength blog — "It's a long road." Hard lessons, strong advice, no bullshit; the title itself a radio call lifted from a Rambo film.

We took the attitude and the cadence of the name — and pointed both at a terminal.
